JavaScript & Ruby - Differences in Looping (Technical blog) 10/24/2014

Loops are an excellent type of logic for automating repetative tasks. Fortunately, they are available in most programming languages in one or more forms. JavaScript and Ruby both support for and while loops, while Ruby additionally supports each, times, upto, downto, and until.

while loops, in Ruby work like so:

puts x = 10
while x > 0

x -= 1

puts x end

while loops, in JavaScript work similarly, but with some immediately noticable differences:

var x = 10
console.log(x)
while (x > 0) {

x -= 1;

console.log(x); }

Functionally, they are exactly the same. They both take the operation of "set the variable x equal to 10, substract 1 from it, display the new value of x, then repeat that over and over while x is greater than 0." If, for example, we had specified the loop to repeat "while x =< 10", the loop would continue forever, essentially causing the computer running it to lock up because while it is subtracting from x, x is always less than or equal to 10. This is called an infinite loop, and requires terminating the program or restarting the computer to terminate it.

The only significant difference here is in syntax. Ruby makes it very easy to set a variable and display it on the console, all in one line, while JavaScript requires a bit more typing. First, the variable has to be explicitly defined with the var keyword. Then, it can only be displayed on the console by a completely separate statement.

JavaScript also requires that the while loop's conditional statement be encapsulated in ()'s, with the actual loop's operations are contained in {}'s. Lastly, each separate operation within the loop must end with a semi-colon. Personally, I prefer Ruby's syntax, as I think entering a new line and ending each statement with a semi-colon is redundant.
