The benefits of version control 09/15/2014
(Taken from my t1-git-blog.md)

The benefits of version control are many. Version control allows multiple developers to contribute to the same project without risk of writing over each others code. It automatically maintains past revisions of code with comments made by the developers describing when and why they made particular changes.

Git helps developers keep track of changes by maintaining separate workspaces for files, with 3 major instances of files: the working directory, the staging area, and the git repo.

Github is an online repository that is incredibly useful for developers to collaborate on projects remotely. It also provides an extensive user-friendly interface to assist with viewing and managing forks and specific code revisions. It even allows code to be made and distributed publicly.
